Guava

Guava is a game engine script (like Lua is to Roblox) that implements a pure Entity Component System (no inheritance, no OOP).

Player.gv external void _init(): Entity player = world.getChild("player") if (player != null and player has MeshRenderer): (player as MeshRenderer).opacity = 0.5

Misc info

The nomenclature for components is to use TitleCase (this means naming Guava files in title case).

Built in functions (ones you should override, like _update(), and ones you probably shouldn't, like _pow(x,p)) are prefixed with _. They are unique to each component; overriding _pow(x,p) in one component won't change its functionality in another.

Static components/functions/variables don't exist. Their functionality can be implemented with singletons (no special logic, just instance a component once and never again).

Traditional "scenes" don't exist. Instead, scenes are treated the same way as component prefabs. To 'switch scenes,' you asynchronously instantiate a component prefab corresponding to the new scene into your current world (disabled by default), disable the component prefab corresponding to the previous scene, enable the new scene, and destroy the old scene.
