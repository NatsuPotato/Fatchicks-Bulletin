Low level, memory safe C alternative. You want to manage raw binary? We got you covered.

The new "bytes" type stores its start and end position in memory, preventing you from performing IO outside of allocated space. Additionally, it stores a "cursor" pointing towards the beginning of its memory. Indexing a bytes returns a new bytes with its cursor shifted. The "<-" operator copies memory to and from bytes.

The "allocate" operator returns a new bytes with the size of the argument (int).

The "sizeof" operator returns the size of the argument (variable or datatype) in bytes as an integer. Remember that different datatypes may have different sizes depending on the OS.

        
bytes data = allocate 1024;
data <- "cats";
data[sizeof "cats"] <- " and dogs";

print("%s", data); // "cats and dogs"
print("%c", data); // 'c'

char first_letter <- data; // 'c'
    

bytes data = allocate 1024;
bytes portion = data[5 int];

// all three of these assignments have the same result
data[sizeof int * 5] <- 20;
data[5 int] <- 20;
portion <- 20;
